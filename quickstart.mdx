---
title: "Get started with OracleTelegramBot Backend"
description: "Configure both Spring Boot services, start them locally, register the Telegram webhook, and send your first KPI command for a live bot response."
---

This guide walks you through everything needed to get OracleTelegramBot Backend running from scratch. By the end you will have both services running locally, the Telegram webhook registered, and a live reply from the bot when you send `/productividad`.

<Steps>

  <Step title="Prerequisites">
    Make sure the following are in place before you start:

    - **Java 17 or later** — the services use Java 17 language features and the `java.time` API.
    - **Apache Maven 3.8+** — used to build and run both services.
    - **Oracle Autonomous Database wallet** — download the wallet zip from your Oracle Cloud console and unzip it to a directory on your machine. You will need the full path to that directory.
    - **Telegram bot token** — create a bot via [@BotFather](https://t.me/BotFather) on Telegram. BotFather will give you a token in the format `123456789:AAF...`.
    - **A public HTTPS URL** — Telegram requires a publicly reachable HTTPS endpoint to deliver webhook updates. Use a tunnel tool such as [ngrok](https://ngrok.com) during local development: `ngrok http 8081`.
  </Step>

  <Step title="Clone the repository">
    Clone the Backend repository and navigate into it:

    ```bash
    git clone https://github.com/OracleTelegramBot/Backend.git
    cd Backend
    ```

    The repository contains two Maven modules: `kpi-service/` and `telegram-service/`.
  </Step>

  <Step title="Configure the KPI service">
    Open `kpi-service/src/main/resources/application.properties` and fill in the values for your environment:

    ```properties kpi-service/src/main/resources/application.properties
    spring.application.name=kpi

    # Full path to the directory where you unzipped the Oracle wallet
    oracle.wallet.path=FULL_PATH

    # TNS alias botbd_tp refers to the connection defined in tnsnames.ora inside the wallet
    spring.datasource.url=jdbc:oracle:thin:@botbd_tp?TNS_ADMIN=${oracle.wallet.path}

    # Oracle database credentials
    spring.datasource.username=CREDENTIALS
    spring.datasource.password=CREDENTIALS

    spring.datasource.driver-class-name=oracle.jdbc.OracleDriver

    # Do not let Hibernate alter the existing schema at startup
    spring.jpa.hibernate.ddl-auto=none

    spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.OracleDialect

    # Log generated SQL to the console for debugging
    spring.jpa.show-sql=true
    ```

    Replace `FULL_PATH` with the absolute path to your unzipped wallet directory (for example `/home/user/wallet/Wallet_botbd`), and replace both `CREDENTIALS` values with your Oracle database username and password.
  </Step>

  <Step title="Configure the Telegram service">
    Open `telegram-service/src/main/resources/application.properties` and update it:

    ```properties telegram-service/src/main/resources/application.properties
    spring.application.name=telegram

    # Base URL of the running kpi-service; change host/port if kpi-service is remote
    kpi.service.url=http://localhost:8080/api/kpis

    # Token provided by BotFather when you created the bot
    telegram.bot.token=TOKEN

    # telegram-service listens on port 8081 so it does not conflict with kpi-service
    server.port=8081
    ```

    Replace `TOKEN` with the token you received from BotFather.
  </Step>

  <Step title="Build and run the KPI service">
    From the `kpi-service/` directory, build the project and start it:

    ```bash
    cd kpi-service
    mvn clean package -DskipTests
    mvn spring-boot:run
    ```

    The service starts on port 8080. You should see a line like `Started KpiApplication in X.XXX seconds` in the console output.

    <Tip>
      Access the interactive API documentation at `http://localhost:8080/swagger-ui.html` once the service is running.
    </Tip>
  </Step>

  <Step title="Build and run the Telegram service">
    Open a second terminal, navigate to `telegram-service/`, and start it:

    ```bash
    cd telegram-service
    mvn clean package -DskipTests
    mvn spring-boot:run
    ```

    The service starts on port 8081. Confirm you see `Started TelegramApplication in X.XXX seconds`.
  </Step>

  <Step title="Register the webhook">
    Telegram needs to know where to send updates. Register your webhook URL using the following curl command — replace `<TOKEN>` with your bot token and `<YOUR_HOST>` with your public HTTPS hostname (for example the ngrok URL):

    ```bash
    curl "https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://<YOUR_HOST>/api/webhook/telegram"
    ```

    A successful registration returns:

    ```json
    {"ok":true,"result":true,"description":"Webhook was set"}
    ```

    Telegram will now POST every incoming message to your `telegram-service` at `/api/webhook/telegram`.
  </Step>

  <Step title="Send your first command">
    Open Telegram, find your bot by the username you chose during BotFather setup, and send:

    ```
    /productividad
    ```

    The bot queries the productivity KPI for project ID 1 and replies with something like:

    ```
    Métrica de Productividad
    Resultado: 75.00%
    Estado: Cálculo de productividad exitoso
    ```

    You can also try `/eficiencia` for the efficiency metric, or send `hola` to see the usage hint.

    <Note>
      Both `/productividad` and `/eficiencia` are hardcoded to query project ID 1. To support other project IDs, you would need to extend the command parsing logic in `telegram-service`.
    </Note>
  </Step>

</Steps>

<Tip>
  The KPI service exposes Swagger UI at `http://localhost:8080/swagger-ui.html` and the Telegram service at `http://localhost:8081/swagger-ui.html`. Use these to explore and test all endpoints interactively without writing any code.
</Tip>
