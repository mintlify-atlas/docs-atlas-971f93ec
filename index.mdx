OracleTelegramBot Backend is a two-service Spring Boot application that lets your team query live project KPIs directly from Telegram. The **KPI service** computes productivity, efficiency, workload, and deadline compliance from an Oracle Autonomous Database. The **Telegram service** handles incoming bot messages and routes commands to the KPI service via OpenFeign, then sends formatted replies back to users.

## How it works

The backend is split into two independent Spring Boot services:

<Steps>
  <Step title="User sends a Telegram message">
    A team member sends `/productividad` or `/eficiencia` to the bot. Telegram delivers the message to the webhook endpoint on the **telegram-service** (port 8081).
  </Step>
  <Step title="Telegram service routes the command">
    The `TelegramServiceImpl` matches the command text and calls the **kpi-service** via a Feign HTTP client on the configured URL.
  </Step>
  <Step title="KPI service queries Oracle">
    The **kpi-service** (port 8080) fetches task data from the `TAREA` table in Oracle Autonomous Database and returns a computed KPI metric.
  </Step>
  <Step title="Bot replies with the result">
    The telegram-service formats the percentage and status message, then POSTs it back to the Telegram API so the user sees the result in chat.
  </Step>
</Steps>

## Key features

  <Card title="Four KPI metrics" icon="chart-line">

  <Card title="Webhook-based bot" icon="bolt">

## Services at a glance

| Service | Port | Purpose |
|---|---|---|
| `kpi-service` | 8080 | Computes and serves KPI metrics from Oracle DB |
| `telegram-service` | 8081 | Receives Telegram webhooks and relays KPI data |
